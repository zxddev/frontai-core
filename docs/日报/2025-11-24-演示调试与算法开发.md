## 三、晚间工作（16:30 - 次日03:00）

### 3.1 工作背景

为周二SSW项目会议准备核心算法模块，进行高强度开发。

### 3.2 完成内容

| 模块 | 功能 | 说明 |
|------|------|------|
| `htn_planner.py` | HTN分层任务网络 | 将抽象任务分解为具体可执行的原子任务序列 |
| `task_decomposer.py` | 任务分解器 | 根据场景识别，加载对应任务链配置 |
| `topological_sort.py` | 拓扑排序 | 基于Kahn算法实现任务依赖排序，生成正确执行顺序 |
| `parallel_scheduler.py` | 并行调度器 | 识别可并行执行的任务组，优化执行效率 |
| `scene_recognizer.py` | 场景识别 | 根据灾情类型识别场景代码（S1、S2、S3等） |
| `chain_merger.py` | 任务链合并 | 合并多条任务链的任务和依赖关系 |
| `mt_library.py` | 元任务库 | 加载和管理元任务配置，提供任务元数据查询 |
| `kg_integration.py` | 知识图谱集成 | 对接Neo4j查询任务依赖和黄金救援时间 |
| `capability_matcher.py` | 能力匹配器 | 任务所需能力与队伍能力的匹配算法 |
| `resource_allocator.py` | 资源分配器 | 基于贪心/NSGA-II的多目标资源优化分配 |
| `constraint_checker.py` | 约束检查器 | 硬规则过滤和软规则评分 |
| `solution_generator.py` | 方案生成器 | 生成多个备选方案供决策 |

### 3.3 核心算法说明

**HTN分层任务网络**：
- 将高层抽象任务（如"地震救援"）分解为具体可执行的原子任务
- 支持多场景组合（地震+次生火灾+危化品泄漏）
- 基于拓扑排序保证任务执行顺序正确
- 识别并行任务组提高救援效率

**多目标优化**：
- 响应时间、覆盖率、成功率、风险、冗余度等多维度优化
- 支持贪心算法和NSGA-II遗传算法两种优化策略
